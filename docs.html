Getting Started

Welcome to BOOT AI! This documentation will help you understand, build, and test the operating system components. The project consists of three main components: KOS (kernel engine), K-DOS (command-line OS), and K-DOS Desktop (GUI layer).

What You Need to Know

  • All projects are in active development (beta/alpha stages)
  • Built primarily with Assembly (x86_64) and C
  • Designed for educational and experimental purposes
  • Requires virtualization software for safe testing

System Requirements

For Development

  • 64-bit x86 processor (Intel or AMD)
  • 4GB+ RAM recommended
  • 10GB free disk space
  • Linux/Unix environment (WSL works on Windows)
  • NASM assembler
  • GCC compiler
  • QEMU or VirtualBox for testing

For Testing

  • VirtualBox, QEMU, or VMware
  • 2GB+ RAM allocated to VM
  • ISO boot capability
⚠️ WARNING: These are experimental operating systems. Do NOT install on physical hardware unless you know what you're doing. Use virtual machines for testing.

KOS Kernel Engine

KOS is the foundational kernel layer that handles boot sequences, memory management, and hardware detection.

Current Features

  • Boot sequence (25-50% complete)
  • Basic memory management
  • Hardware detection and driver framework
  • Minimal overhead architecture

Quick Test

# Download beta ISO
wget https://github.com/MGKFN/BOOTAI/releases/download/OS/kdos.2.iso

# Run with QEMU
qemu-system-x86_64 -cdrom kdos.2.iso -m 512M

# Or with VirtualBox
# Create new VM, attach ISO, boot

K-DOS Command-Line OS

K-DOS provides a terminal-based operating environment with basic file system operations.

Available Commands

  • ls - List directory contents
  • cd [dir] - Change directory
  • mkdir [name] - Create directory
  • help - Show available commands
  • clear - Clear screen

System Info

  • Runs entirely from RAM
  • Fast boot time (~2-3 seconds in VM)
  • No persistent storage yet (coming soon)

K-DOS Desktop (Alpha)

Early-stage graphical user interface built on top of K-DOS. Currently in active development (10-25% complete).

Roadmap

  1. Phase 1 (Current): Basic window rendering, framebuffer graphics
  2. Phase 2: Input handling, mouse support
  3. Phase 3: Window manager, UI components
  4. Phase 4: Application framework
⚠️ ALPHA SOFTWARE: K-DOS Desktop is highly experimental and may crash or behave unexpectedly. Use only for testing purposes.

Build Instructions

Want to build from source? Here's how to compile the projects yourself.

Prerequisites

# Ubuntu/Debian
sudo apt update
sudo apt install nasm gcc make qemu-system-x86

# Arch Linux
sudo pacman -S nasm gcc make qemu

# macOS
brew install nasm gcc qemu

Building KOS/K-DOS

# Clone repository (example)
git clone https://github.com/MGKFN/BOOTAI
cd BOOTAI

# Build kernel
make kernel

# Create bootable ISO
make iso

# Test in QEMU
make run

Troubleshooting

  • If build fails: Check that all dependencies are installed
  • If QEMU won't boot: Verify ISO was created successfully
  • For errors: Check the blog for known issues

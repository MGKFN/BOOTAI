🚀 Getting Started

System Requirements

Minimum:

  • x86_64 processor (Intel or AMD)
  • 2GB RAM
  • 10GB disk space
  • VGA graphics adapter
  • UEFI or BIOS boot support
⚠️ Warning: BOOT AI is experimental software. Always test in a virtual machine first. Do not install on hardware with important data.

Quick Start Guide

  1. Download the latest ISO from the Products page
  2. Create a new VM with 4GB RAM and 20GB disk
  3. Mount the ISO as a boot device
  4. Start the VM and follow on-screen prompts
  5. Default login: root (no password in beta builds)

🔨 Building from Source

Prerequisites

  • GCC cross-compiler for x86_64-elf target
  • NASM assembler (version 2.14+)
  • Make build system
  • QEMU for testing (optional but recommended)

Build Commands

git clone https://github.com/MGKFN/BOOTAI.git cd BOOTAI make kernel make kdos make iso make run

🔧 Troubleshooting

System won't boot / Kernel panic

  • Verify VM has at least 2GB RAM allocated
  • Ensure UEFI/BIOS boot is properly configured
  • Try disabling SMP in BIOS if using multi-core

Desktop crashes on load

  • Desktop is alpha quality — expect crashes
  • Verify framebuffer support in your VM
  • Try reducing screen resolution to 800x600

Debug Mode

boot: kdos debug=1 loglevel=7

Debug messages will appear on serial port COM1 (115200 baud).

🆕 BOOTOS — Getting Started

What is BOOTOS?

BOOTOS is a brand new operating system project in the BOOT AI ecosystem. Unlike KOS which uses a custom bootloader, BOOTOS is built on the Multiboot2 specification and uses GRUB as its bootloader. This makes it easier to test on real hardware and provides a standardized boot interface right from the start.

How Multiboot2 Works

Multiboot2 is an open standard that defines how a bootloader (like GRUB) loads an operating system kernel. When GRUB loads a Multiboot2-compliant kernel, it passes a structured information table to the kernel entry point containing:

  • Memory map — what RAM is available and where
  • Framebuffer info — screen resolution, mode, and address
  • Boot modules — additional files loaded alongside the kernel
  • Command line arguments — parameters passed at boot

Setting Up GRUB for BOOTOS (When Available)

# Install GRUB (Linux development host) sudo apt install grub2 xorriso # GRUB config entry (grub.cfg) menuentry "BOOTOS" { multiboot2 /boot/bootos.elf boot }

Multiboot2 Kernel Header (C)

#define MB2_MAGIC 0xe85250d6 #define MB2_ARCH 0 #define MB2_LENGTH sizeof(header) #define MB2_CHECKSUM -(MB2_MAGIC + MB2_ARCH + MB2_LENGTH) struct multiboot2_header { uint32_t magic; uint32_t architecture; uint32_t header_length; uint32_t checksum; };

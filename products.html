KOS Kernel Engine

Technical Architecture

KOS is a bare-metal kernel written in x86_64 Assembly and C following a microkernel design philosophy. The boot sequence initializes hardware in under 3 seconds on modern systems using UEFI or legacy BIOS.

Current Features

  • 16/32/64-bit x86 support with UEFI and BIOS boot
  • 4KB/2MB/1GB page size support with buddy allocator
  • AHCI/NVMe storage drivers with DMA
  • PS/2 and USB keyboard/mouse input
  • Experimental SMP multi-core scheduler

Roadmap

Q2 2026: Complete multi-core scheduling | Q3 2026: TCP/IP network stack | Q4 2026: User-mode transition and ELF loader

K-DOS Operating System

Technical Architecture

K-DOS is a UNIX-like command-line OS built on KOS. It runs entirely from RAM with an 8MB footprint, making it ideal for embedded systems and testing. All core commands are built-in to avoid filesystem dependencies during early boot.

Available Commands

  • ls, cd, mkdir, rmdir — filesystem navigation
  • cat, cp, mv — file operations
  • ps, kill — process management
  • help, clear, reboot, shutdown — system commands

Roadmap

Q2 2026: Shell scripting and pipes | Q3 2026: Package manager | Q4 2026: Network commands and persistent storage

K-DOS Desktop Environment

Technical Architecture

K-DOS Desktop is a custom GUI layer using direct framebuffer rendering for maximum performance. The stacking window manager handles input through a kernel-level event system translated into GUI interactions.

Current Progress

  • Basic window rendering with title bars (✓ Complete)
  • Mouse cursor with custom themes (✓ Complete)
  • Window dragging and resizing (⚠ In Progress)
  • Context menus and taskbar (⏳ Planned)

Roadmap

BOOTOS — New Project

What is BOOTOS?

BOOTOS is a brand new operating system project built from the ground up using the Multiboot2 specification and the GRUB bootloader. Unlike KOS which uses a custom bootloader, BOOTOS leverages GRUB's mature, widely-supported boot infrastructure for broader hardware compatibility right from the start.

Why Multiboot2 & GRUB?

  • Industry-standard boot protocol supported by virtually all x86 hardware
  • GRUB handles complex boot environments so the kernel can focus on OS functionality
  • Multiboot2 provides a standardized interface between bootloader and kernel
  • Easier to test on real hardware without custom bootloader setup
  • Built-in support for modules, memory maps, and framebuffer info

Technical Architecture

BOOTOS will be written in C with x86_64 Assembly for low-level hardware interaction. The Multiboot2 header allows GRUB to load the kernel and pass essential system information including memory layout, available hardware, and display configuration directly to the kernel entry point.

Current Status

Early development — 0-10% complete. Basic Multiboot2 kernel entry is the current focus. No download available yet. Follow the blog and Discord for updates.

Roadmap

Phase 1: Multiboot2 kernel entry + VGA text output
Phase 2: Memory management + interrupt handling
Phase 3: Basic shell and filesystem
Phase 4: GRUB menu integration for multi-OS boot

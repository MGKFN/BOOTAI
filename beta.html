KOS 0.2.9 Beta

Installation Guide

Requirements: x86_64 CPU, 2GB RAM, UEFI or BIOS boot, VM recommended (VMware/VirtualBox).

  • Download kdos.2.iso
  • Create VM: 4GB RAM, 20GB disk
  • Mount ISO as boot device and start VM
  • Default login: root (no password)

Known Issues

  • SMP unstable on more than 4 cores
  • USB 3.0 not fully supported
  • No network stack yet

Version History

0.2.9: SMP scheduler, buddy allocator | 0.2.5: AHCI driver, PS/2 input | 0.2.0: Initial beta

K-DOS 1.0 Beta

Installation Guide

Requirements: KOS 0.2.9+ kernel, 512MB RAM, terminal access.

  • Download kernel.img
  • Boot KOS first, then: "load kdos kernel.img"
  • Type "help" for available commands

Known Issues

  • Filesystem RAM-only, no persistence
  • No pipe or redirection support
  • Tab completion missing

Version History

1.0 Beta: cat, ps, kill commands | 0.8: Directory ops, file I/O | 0.5: Initial shell

K-DOS Desktop Alpha

Installation Guide

Requirements: K-DOS 1.0+, 1GB RAM, framebuffer graphics, USB or PS/2 mouse.

  • Download kosIDK.img
  • From K-DOS shell: "startx kosIDK.img"
  • Desktop loads in 30–60 seconds

Known Issues

  • Window dragging loses mouse sync
  • No close/minimize buttons yet
  • Cursor disappears on some VMs

Version History

Alpha 0.3: Window rendering, mouse | Alpha 0.2: Framebuffer, cursor | Alpha 0.1: Static window display

BOOTOS — Project Information

What is BOOTOS?

BOOTOS is a brand new operating system built using the Multiboot2 specification and GRUB bootloader. It is the most recent project in the BOOT AI ecosystem and takes a different approach from KOS by using industry-standard boot protocols from day one.

Why Multiboot2 & GRUB?

  • GRUB is supported by virtually all x86 hardware out of the box
  • Multiboot2 gives the kernel a standardized way to receive memory maps, framebuffer info and module data from the bootloader
  • No need to write and maintain a custom bootloader — GRUB handles the hard parts
  • Easier to test on real physical hardware from the start

Current Status

0-10% complete. Working on the basic Multiboot2 kernel entry point and VGA text output. No downloadable build yet.

How to Get Notified

Join the Discord server to get pinged when the first alpha build is released. Updates will also be posted on YouTube and in the blog.

Planned Roadmap

  • Phase 1: Multiboot2 entry + VGA text output
  • Phase 2: Memory management + interrupts
  • Phase 3: Basic shell and filesystem
  • Phase 4: GRUB menu for multi-OS boot

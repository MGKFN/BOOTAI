Tech Haven Studios — KOS™ v0.2.9 Beta

KOS™ 0.2.9 Beta

Installation Guide

Requirements: x86_64 CPU (Intel or AMD), 2GB RAM minimum, UEFI or BIOS boot support. Recommended: VMware Workstation or VirtualBox with 4GB RAM and 20GB disk.

  • Download kdos.2.iso from the button above
  • Create new VM: 4GB RAM, 20GB disk, 64-bit
  • Mount ISO as boot device and power on
  • Default login: root (no password in beta)

Known Issues

  • SMP unstable on systems with more than 4 cores
  • USB 3.0 devices not fully supported yet
  • AHCI driver may fail on some NVMe setups
  • No network stack yet — networking non-functional

System Requirements

Minimum: x86_64, 2GB RAM, 10GB disk, VGA graphics
Recommended: Multi-core x86_64, 4GB+ RAM, 20GB+ disk, UEFI boot

Version History

0.2.9: SMP scheduler, buddy allocator rewrite, PCI fixes
0.2.5: AHCI driver, PS/2 keyboard and mouse
0.2.0: Initial beta — boot sequence and memory management

Tech Haven Studios — K-DOS™ v1.0 Beta

K-DOS™ 1.0 Beta

Installation Guide

Requirements: KOS 0.2.9+ kernel running, 512MB RAM minimum, terminal or serial console access.

  • Download kernel.img from the button above
  • Boot KOS kernel first
  • From KOS prompt: load kdos kernel.img
  • System boots into K-DOS shell
  • Type help to see all available commands

Known Issues

  • Filesystem is RAM-only — no persistent storage yet
  • No pipe or I/O redirection support
  • Tab completion not yet implemented
  • Process listing may show incorrect memory values

System Requirements

Minimum: KOS kernel, 512MB RAM, terminal display
Recommended: 1GB+ RAM, 80x25 terminal resolution minimum

Version History

1.0 Beta: cat, ps, kill commands — improved shell parsing
0.8 Beta: Directory operations, basic file I/O
0.5 Beta: Initial shell with ls, cd, mkdir, help

Tech Haven Studios — K-DOS Desktop™ Alpha

K-DOS Desktop™ Alpha

Installation Guide

Requirements: K-DOS 1.0+ running, 1GB RAM minimum, framebuffer graphics support, USB or PS/2 mouse required.

  • Download kosIDK.img from the button above
  • Boot K-DOS system first
  • From K-DOS shell: startx kosIDK.img
  • Desktop environment loads (may take 30–60 seconds)
  • Use mouse to interact — keyboard input is limited

Known Issues

  • Window dragging occasionally loses mouse sync
  • No window close or minimize buttons yet
  • Rendering artifacts visible when windows overlap
  • Mouse cursor may disappear on certain VM configurations
  • Keyboard input unreliable in GUI mode
  • No applications available yet — window manager demo only

System Requirements

Minimum: K-DOS, 1GB RAM, 800x600 framebuffer, PS/2 or USB mouse
Recommended: 2GB+ RAM, 1024x768+ resolution, USB mouse

Version History

Tech Haven Studios — BOOTOS™

BOOTOS™ — Project Info

What is BOOTOS?

BOOTOS is a brand new operating system built using the Multiboot2 specification and the GRUB bootloader. Unlike KOS which uses a custom bootloader, BOOTOS leverages GRUB's mature, widely-supported boot infrastructure for maximum hardware compatibility from day one.

Why Multiboot2 and GRUB?

  • GRUB supports virtually all x86 hardware ever manufactured
  • Multiboot2 provides a standardized interface between bootloader and kernel
  • Handles UEFI and legacy BIOS automatically
  • No custom bootloader to maintain — focus stays on the OS
  • Built-in support for multi-OS GRUB boot menus

Current Status

BOOTOS is 0-10% complete. Active development focus is on the Multiboot2 kernel entry point and basic VGA text output. No downloadable build is available yet.

What is BOOT-DOS?

BOOT-DOS is an exclusive new DOS-style operating system from Tech Haven Studios. It is a spiritual successor to K-DOS — taking the familiar terminal-first interface further with expanded commands, improved filesystem support, better hardware compatibility, and new features not found in K-DOS.

Why Discord-Exclusive?
